ASP.NET __doPostBack Test

Problem Explanation:

In ASP.NET WebForms, __doPostBack() calls form.submit() programmatically.

Critical Issue: form.submit() does NOT trigger the submit event listener!

This means the FormProof script's submit handler never fires when using __doPostBack.

Test 1: Normal Submit (Works)
Test 2: __doPostBack (Problem)
Test 3: Solution - Using onsubmit attribute (Recommended for ASP.NET)

Solution for ASP.NET Integration:

To make FormProof work with ASP.NET __doPostBack, you need to either:

  1. Override __doPostBack - Intercept the function and dispatch a submit event before calling the original
  2. Use form.onsubmit attribute - ASP.NET respects the onsubmit attribute even with programmatic submit
  3. Hook into requestSubmit() - Modern browsers support this which DOES trigger the submit event
